# 장기 UI/UX 고도화 추진 계획서 (Commercial-Grade Roadmap)

현재의 모바일 대시보드는 8-bit 오디오 프레임워크와 스플래시 오버레이, 네비게이션 뱃지 등을 도입하여 **'기능적으로 완성된 몰입형 웹 인터페이스'**의 기준은 충족했습니다. 그러나 실제 상용 마피아 게임(Mafia42 등)이 주는 원초적인 '손맛'과 '시각적 카타르시스'에 도달하기 위해서는 다음과 같은 핵심적인 한계점들이 존재합니다.

## 1. 현재 UI/UX의 한계점 (평가)

1. **정적 렌더링 및 이펙트 부재 (Lack of Game Juice)**
   - **평가**: 현재 상태가 변할 때 CSS 애니메이션으로 팝업을 띄우고 사운드를 트는 수준입니다. 마피아가 총을 쏠 때 화면이 흔들리거나(Screen Shake), 핏자국이 튀는 파티클 이펙트, 밤이 될 때 안개가 끼는 시각적 효과가 없습니다.
   - **체감 하락 원인**: 텍스트 채팅과 아이콘만으로는 처형이나 밤의 살해 이벤트가 주는 극적인 텐션을 살리지 못합니다.
2. **HTTP 폴링 아키텍처의 한계 (Latency & Sync Drift)**
   - **평가**: 현재 클라이언트는 일정 주기마다 서버의 API를 호출(Polling)하여 상태를 가져옵니다. 
   - **체감 하락 원인**: 실시간 채팅의 즉각적인 반응성(10ms 이내)이나, 남은 10초 타이머의 정확한 째깍거림 오차를 완전히 제어하기 힘듭니다. 
3. **제스처 및 마이크로 인터랙션 부족 (Micro-interactions & Gestures)**
   - **평가**: 하단의 Dock 메뉴 터치로만 뷰를 전환합니다.
   - **체감 하락 원인**: 최신 모바일 앱에서 흔히 사용하는 '스와이프(Swipe)'를 통한 탭 전환이나, 채팅 말풍선이 자연스럽게 밀려 올라오는 스무스 스크롤링 애니메이션이 비어있어 네이티브 앱 같은 쫀쫀함이 부족합니다.
4. **접근성 및 설정 인터페이스 (Accessibility & Settings)**
   - **평가**: 오디오 볼륨을 유저가 앱 내에서 개별 조절하거나 끄는 설정창이 없습니다.

## 2. 장기 추진 계획 및 기술 로드맵 (Long-Term Roadmap)

본 계획은 웹의 가벼움을 유지하되, 캔버스(Canvas) 기반 이펙트와 소켓 통신을 도입하여 **'100% 상용 게임 수준'**으로 도약하기 위한 로드맵입니다.

### Phase 1: 이벤트 시퀀스 연출 (Visual Sequence & Particles)
상태 변화가 일어나는 찰나의 순간을 애니메이션 시퀀스로 구축합니다.
- **Screen Shake API**: 투표 처형, 마피아 타겟팅 완료 등의 결정적 순간에 CSS/JS 기반 화면 흔들림 효과 적용.
- **Particle System 도입**: 초경량 Canvas JS (또는 CSS 파티클)를 활용하여 낮->밤 전환 시 안개 이동, 처형 시 사선(Slash) 애니메이션 동적 렌더링.
- **동적 아바타**: 유저석(Seat) 상태를 정적 CSS 카드에서 타겟팅 당할 시 깜빡이거나 숨쉬는(Breathing) 듯한 CSS 트랜스폼 애니메이션 적용.

### Phase 2: 코어 엔진 WebSocket 마이그레이션 (Realtime Sync)
HTTP Polling 방식을 폐기하고, 완전한 실시간 양방향 통신으로 렌더링을 교체합니다.
- **빠른 반응성**: 채팅 전송과 동시에 다른 클라이언트 화면에 말풍선이 Bounce 애니메이션과 함께 등장.
- **서버 타임 동기화**: `requestAnimationFrame`과 서버 Time offset을 정밀하게 계산하여 모든 플레이어가 정확히 0.01초 단위까지 동일한 카운트다운을 보도록 개선.

### Phase 3: 모바일 네이티브 UX/Gesture 편입
웹 브라우저의 한계를 넘는 조작감을 구현합니다.
- **스와이프 탭 전환**: 화면 좌우 스와이프로 '개인행동 - 상태 - 공개채팅 - 비밀채팅' 간 매끄러운 화면 슬라이드 전환.
- **Bottom Sheet 고도화**: 현재 팝업 형태인 추리 메모장(Memo) 등을 드래그해서 내릴 수 있는(Drag-to-dismiss) 네이티브 Bottom Sheet 컴포넌트로 업그레이드.
- **인게임 설정 (Settings)**: BGM / SFX 개별 볼륨 슬라이더 및 시각 효과 On/Off 토글 지원.

### Phase 4: 게임 결과 / 컷씬 (Cinematic Ending)
게임이 끝났을 때 밋밋한 텍스트 카드 대신 영화 같은 컷씬을 제공합니다.
- **승리/패배 씬**: 살아남은 자들의 캐릭터 아이콘이 순차적으로 도장(Stamp) 찍히듯 터지며 성과 통계(누가 누구를 죽였는지 킬 로그 타임라인) 제공.

> **결론**: 현재는 "완벽히 잘 작동하는 프로토타입 / 고품질 웹 대시보드" 단계입니다. 장기 계획(Canvas 이펙트, WebSocket, 제스처, 연출 시퀀스)이 달성될 경우, 외부 스토어에 웹앱(PWA/Webview)으로 즉시 출시해도 손색없는 상용 마피아 게임 UX를 확보할 수 있습니다.
